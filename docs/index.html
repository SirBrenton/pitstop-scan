Pitstop Scan

“It usually works… but the tail is brutal.”

Drop in a small JSONL receipt file — get a ranked fix order in ~5 minutes.

If retries are spiking, 429s are showing up, or you can’t tell which call is killing your p99 — Pitstop Scan shows where the pain concentrates and what to cap first.

  • Which calls are burning retry budget
  • Where latency budgets are breached — even on calls that returned “ok”
  • What to cap first: deadlines, retry caps, backoff rules, cooldown scope
✓ Local-only by default.
✓ Receipts contain no prompts, payloads, headers, tokens, or raw URLs.
✓ Normalized metadata only.
